The ValidatorÂ¶

Validation is a very common task in web applications. Data entered in forms needs to be validated. Data also needs to be validated before it is written into a database or passed to a web service.

Symfony2 ships with a Validator component that makes this task very easy. This component is based on the JSR303 Bean Validation specification. What? A Java specification in PHP? You heard right, but it’s not as bad as it sounds. Let’s look at how we use it in PHP.

The validator validates objects against constraints. Let’s start with the simple constraint that the $name property of a class Author must not be empty:

// Sensio/HelloBundle/Author.php
class Author
{
    private $name;
}

The next listing shows the configuration that connects properties of the class with constraints; this process is called the “mapping”:

Finally, we can use the Symfony\Component\Validator\Validator class for validation. To use the default Symfony2 validator, adapt your application configuration as follows:

# hello/config/config.yml
framework:
    validation:
        enabled: true

Now call the validate() method on the service, which delivers a list of errors if validation fails.

$validator = $container->get('validator');
$author = new Author();

print $validator->validate($author);

Because the $name property is empty, you will see the following error message:

Sensio\HelloBundle\Author.name:
    This value should not be blank

Insert a value into the property and the error message will disappear.
